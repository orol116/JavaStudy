변수 선언 방식

        var   : 변수, 변수명 중복 O (덮어쓰기 가능), 함수 레벨 scope
        let   : 변수, 변수명 중복 X, 블록{} 레벨 scope
        const : 상수, 변수명 중복 X, 블록{} 레벨 scope

        1순위 : const
                (JS는 특정 요소를 선택해서 사용하는 경우가 많아서 변수에 고정시켜둔다.)

        2순위 : let

        3순위 : var (ES6 이후부터 사용 빈도가 많이 적어졌다.)

자바스크립트의 자료형

        자바스크립트는 변수 선언 시 별도의 자료형을 지정하지 않는다.
        -> 번수에 대입되는 값(리터럴)에 의해서 자료형이 결정된다.

        - string (문자열)
        - number (숫자)   -> 정수/실수 모두 포함
        - boolean (논리값)
        - object (객체)   -> 배열(Array), 자바스크립트 객체 {k:v, k:v}
        - function (함수)
        - undefined (정의되지 않은 변수) -> 변수가 선언만 됐지, 값이 대입되지 않았다.

        (참고)
        - null (참조하는게 없음) -> document.getElementById("오타")
                                 -> id가 일치하는 요소가 없어서 참조할 수 없다.
                                 -> null 은 타입보단 값(리터럴)의 개념이다.
